Key Mental Model

★ Insight ───────────────────────────────────── - Worktree = folder — it's where you edit files - Branch = pointer to a commit — it tracks what version you're on - Remote branch = GitHub's copy of that pointer — synced via push/pull - One worktree per branch — you can't have two worktrees on the same branch ─────────────────────────────────────────────────

The Big Picture

Worktrees are local convenience. Branches are the real deal.

When you graduate an experiment to its own repo, you're essentially: 1. Taking one branch's history 2. Pushing it to a new GitHub repo 3. Removing the worktree (because it's now a separate repo, not part of helloworld)
